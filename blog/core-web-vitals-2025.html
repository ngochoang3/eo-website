Core Web Vitals 2025: Những thứ thực sự ảnh hưởng đến ranking Google

5 tháng 6, 2025·7 phút đọc·EO Technology

Google chính thức đưa Core Web Vitals vào ranking signal từ tháng 6/2021 (Page Experience update). Từ tháng 3/2024, INP (Interaction to Next Paint) thay thế FID. Bài này tổng hợp trạng thái hiện tại của 3 Core Web Vitals và kỹ thuật optimize thực tế — không lý thuyết.

Nguồn: Tất cả ngưỡng và thông tin trong bài dựa trên web.dev/vitals và Google Search Central.

3 Core Web Vitals hiện tại (2024–2025)

LCP — Largest Contentful Paint

Thời gian để element lớn nhất trên viewport (thường là hero image hoặc heading lớn) được render. Đây là metric phản ánh tốc độ tải trang từ góc nhìn người dùng.

Tốt: ≤ 2.5s Cần cải thiện: 2.5s – 4.0s Kém: > 4.0s

CLS — Cumulative Layout Shift

Tổng cộng "layout shift" không mong đợi trong suốt vòng đời trang — khi element dịch chuyển đột ngột (ví dụ: quảng cáo load xong đẩy content xuống). Đơn vị là điểm, không phải giây.

Tốt: ≤ 0.1 Cần cải thiện: 0.1 – 0.25 Kém: > 0.25

INP — Interaction to Next Paint (thay FID từ 3/2024)

Latency của tất cả user interactions (click, tap, keyboard) trong suốt vòng đời trang — lấy percentile thứ 75 cao nhất. Khác FID chỉ đo lần interaction đầu tiên, INP đo toàn bộ session.

Tốt: ≤ 200ms Cần cải thiện: 200ms – 500ms Kém: > 500ms

8 kỹ thuật optimize áp dụng được ngay

1. Preload LCP image

Nếu LCP element là image, thêm <link rel="preload" as="image"> vào <head>. Image sẽ được tải song song với HTML parsing thay vì đợi đến khi render tree build xong. Thường cải thiện LCP 300-800ms.

2. Dùng fetchpriority="high" cho hero image

Thêm thuộc tính fetchpriority="high" vào thẻ <img> của LCP element. Browser sẽ ưu tiên tải image này trước. Kết hợp với preload cho hiệu quả tốt nhất.

3. Đặt kích thước cố định cho image và embed

Luôn đặt width và height cho <img>, <video>, <iframe>. Browser sẽ reserve space trước khi content load, tránh layout shift. Đây là fix đơn giản nhất cho CLS cao.

4. Tránh chèn content trên existing content

Không inject banner, cookie consent, hay dynamic content phía trên content đã render trừ khi đã reserve space bằng min-height. Font loading cũng gây CLS — dùng font-display: optional hoặc preload font.

5. Break up long tasks trên main thread

INP cao thường do long tasks (tasks >50ms) block main thread. Dùng scheduler.yield() (hoặc setTimeout(0) nếu browser cũ) để chia nhỏ JS task, cho browser có cơ hội xử lý input.

6. Lazy load non-critical JS

Script không cần thiết ngay khi load trang → dùng defer hoặc dynamic import. Giảm Total Blocking Time giúp INP và LCP cùng cải thiện. Đặc biệt quan trọng với third-party scripts (analytics, chat widget).

7. Dùng CDN và cache headers đúng cách

Static assets (JS, CSS, image) cần Cache-Control: max-age=31536000, immutable với content-hashed filename. Dynamic HTML cần Cache-Control: no-store hoặc stale-while-revalidate. CDN giảm TTFB, TTFB thấp giúp LCP.

8. Image format hiện đại: WebP và AVIF

AVIF nhỏ hơn WebP ~20%, WebP nhỏ hơn JPEG ~30% ở cùng chất lượng. Dùng <picture> element với fallback. Next.js, Nuxt, Astro đều tự động optimize — nếu dùng framework khác, cần cài sharp hoặc imagemin vào build pipeline.

Core Web Vitals có thực sự ảnh hưởng đến ranking?

Có, nhưng đây là một trong rất nhiều ranking signals. Google xác nhận CWV là "tiebreaker" — khi 2 trang có content quality tương đương, trang có CWV tốt hơn sẽ được ưu tiên. Đừng kỳ vọng chỉ cải thiện CWV sẽ tăng 10 bậc nếu content không đủ tốt.

Nơi CWV có tác động rõ ràng nhất: Google Discover và AMP — nơi page experience signal có weight cao hơn.

Cách đo Core Web Vitals

  • Google Search Console → Core Web Vitals report: dữ liệu thực từ Chrome users (Field data)
  • PageSpeed Insights (pagespeed.web.dev): cả Lab data và Field data
  • Chrome DevTools → Performance tab: Lab measurement cho từng session
  • web-vitals npm package: embed vào app để thu thập real user data

Quan trọng: Chỉ dùng Field data (real user data) để đánh giá impact đến SEO. Lab data (PageSpeed score) hữu ích để debug nhưng không phản ánh trải nghiệm thực tế của user.
